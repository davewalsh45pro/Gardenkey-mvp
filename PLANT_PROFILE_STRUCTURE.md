# GardenKey Plant Profile Structure v1.0

Use this structure before creating lots of plant pages. The aim is to keep every GardenKey profile consistent, easy to scan, and useful on a phone.

## 1. Plant Identity
- GardenKey ID
- Common name
- Latin name
- Plant type
- Photo placeholder
- Owner / location note, optional

## 2. Quick Care Summary
- Light requirement
- Watering requirement
- Feeding requirement
- Soil / compost preference
- Hardiness / shelter requirement
- Pruning or maintenance action

## 3. Seasonal Actions
- Spring
- Summer
- Autumn
- Winter

## 4. Personal Log
- Last watered
- Last fed
- Last repotted
- Last pruned
- Notes / observations

## 5. Issues & Troubleshooting
- Common pests
- Common diseases
- Warning signs
- What to do first

## 6. Companion / Placement Notes
- Good companions
- Avoid planting near
- Balcony / container suitability
- Wildlife value

## 7. Links
- Supplier link
- Care guide link
- RHS or trusted reference link
- Owner notes link, optional

## MVP Rule
Do not overbuild the first version. For testing, the page only needs to answer:

1. What plant is this?
2. What does it need right now?
3. What should I do this season?
4. Can I record a simple note?
